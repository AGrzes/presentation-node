Getting Started

User Installation

Avoid sudo when managing packages

Download

Node.js

/home/{user}/opt/{node}

Set up path in .bashrc

PATH=/home/{user}/opt/{node}/bin:$PATH export PATH

Set up path in .npmrc

root = /home/{user}/opt/{node}/lib/node_modules binroot = /home/{user}/opt/{node}/bin manroot = /home/{user}/opt/{node}/share/man

Run

> echo "console.log('hello world');" > hello.js && node hello.js

Programming model

JavaScript

  • ECMAScript
  • C - like syntax
  • NOT - Java
  • Types - Dynamic
  • Objects - Prototype based
  • Functions - first class entities
  • Watch out - "this", scoping, equality ...

Single threaded

  • No thread safety issues
  • Heavy lifting will block entire program
  • To scale - run multiple processes
  • To synchronize - IPC

Asynchronous / Event Based

  • Do not block the process waiting for result
  • Register success / failure handler and deal with asynchronous programming
  • Much of API designed in asynchronous model

Modules

A way to organize code

Declaring module

Use exports variable to define module public API. Any property assigned to it will be accessible to module clients. module.exports is the same as exports but you can assign it whole

exports

// my-module.js exports.a = 7; exports.b = "text"; exports.c = function(param){ return param * 2 ;};

module.exports

// my-module.js module.exports = { a : 7, b : "text", c : function(param){ return param * 2 ;} };

Importing module

Use require function to load module

Usage

// my-other-module.js var myModule=require("./my-module.js"); console.log(myModule.a); // 7 console.log(myModule.b); // text console.log(myModule.c(7)); // 14

Lookup

  • "./path/to/module/file/js"
  • "/path/to/module"
  • "module-name"

For named modules the lookup search for module-name in ./node_modules directory then in ../node_modules directory, then ../../node_modules and so on until reaching filesystem root and then inside global node_modules directory.

NPM

the package manager for ... (Node.js)

Install module

  • >npm install ${module} - Install in the current directory
  • >npm install -g ${module} - Install globally
  • >npm install --save ${module} - Install and add to current module dependencies

package.json

Describe current module and all of its dependencies.

>npm install in module directory installs all declared dependencies.

scripts

Defined in package.json allow to describe lifecycle operations for the module.

... "scripts": { "start": "node app.js" }, ... >npm start

bin

Defined in package.json allow to register global commands.

... "bin": { "do-something": "./module.js" }, ... > npm install -g > do-something with arguments

The package have to me installed globally for command to work.

npm link

> npm link creates symbolic link from global modules directory to current module

> npm link ${module} creates symbolic link from module in global modules to local node_modules

Useful for development purpose - to instantly see changes in linked module without reinstallation.

Promises

  • Asynchronous action return promise
  • Promise have then and catch method to register result handlers
  • then and catch methods return new promises - for the results of the handlers - enabling promise chaining
  • Handlers can return values which will be resolved immediately or another deferred promises
doSomething("with","arguments").then(function(success){ // handle success }).catch(function(error){ // handle error })

Summary

Why Use Node?

  • Same language on server and on the client (and sometimes in database)
  • Sharing code between application layers
  • No build required
  • Minimal development setup
  • Available cloud hosting
  • Rich module ecosystem

Use cases

  • Web application
  • Communication application
  • Utility script

Why Not Use Node?

  • Heavy computations
  • Lack of specific libraries for the task
  • Lack of JavaScript skills
  • Not fit in existing architecture
